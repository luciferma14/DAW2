Referencias de instrucciones y consultas SQL

Comandos Básicos

Nivel Base de Datos (Database):

Enlace Descripción
CREATE DATABASE Crea una nueva base de datos en el sistema de gestión de bases de datos.
DROP DATABASE Elimina permanentemente una base de datos
BACKUP DATABASE Realiza una copia de seguridad completa de una base de datos (en sistemas que lo soportan).

Nivel Tabla (Table):

Enlace Descripción
CREATE TABLE Crea una nueva tabla dentro de una base de datos con sus columnas y tipos de datos.
DROP TABLE Elimina una tabla existente junto con todos sus datos.
ALTER TABLE Modifica la definición de una tabla: agregar, eliminar o cambiar columnas o restricciones.

Nivel Registro o Datos (Data):

Enlace Descripción
SELECT Consulta para extraer datos de una o más tablas.
SELECT DISTINCT Devuelve sólo los valores distintos (sin duplicados) de la(s) columna(s) seleccionada(s).
WHERE Filtro para limitar las filas que cumplen una condición específica dentro de SELECT/UPDATE/DELETE.
ORDER BY Ordena el resultado de una consulta por una o más columnas (ascendente o descendente). :contentReference[oaicite:8]{index=8}
AND / OR / NOT Operadores lógicos para combinar condiciones en cláusulas WHERE u otras condiciones.
INSERT INTO Inserta nuevos registros o filas en una tabla con valores para sus columnas.
UPDATE Modifica uno o más registros existentes en una tabla, según condiciones definidas.
DELETE Elimina uno o más registros de una tabla según una condición especificada en la cláusula WHERE.

Ejemplos SQL — Tablero Kanban

Este documento muestra ejemplos prácticos de comandos SQL aplicados a una base de datos que gestiona un tablero Kanban, con una tabla principal tasks.

1. Base de Datos

🔹 Crear la base de datos

CREATE DATABASE kanban_board;

🔹 Seleccionar la base de datos

USE kanban_board;

🔹 (Opcional) Hacer copia de seguridad

-- Para MySQL/MariaDB
mysqldump -u root -p kanban_board > kanban_board_backup.sql

-- Para restaurar:
mysql -u root -p kanban_board < kanban_board_backup.sql

🔹 (Opcional) Eliminar la base de datos

DROP DATABASE kanban_board;

2. Tablas

🔹 Crear la tabla principal tasks

CREATE TABLE tasks (
    id INT AUTO_INCREMENT PRIMARY KEY,
    description TEXT,
    status ENUM('deleted',
                'Some day', 
                'This week', 
                'Tomorrow', 
                'Today', 
                'In progress',
                'Done') DEFAULT 'Some day',
    priority ENUM('top',
                  'high', 
                  'medium', 
                  'low') DEFAULT 'medium',
    created_at TIMESTAMP 
               DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP 
               DEFAULT CURRENT_TIMESTAMP 
               ON UPDATE CURRENT_TIMESTAMP
);

🔹 Modificar la tabla (agregar columna)

ALTER TABLE tasks ADD COLUMN owner VARCHAR(50);

🔹 Eliminar la tabla

DROP TABLE tasks;

3. Consultas y Manipulación de Datos

🔹 Insertar tareas

INSERT INTO tasks (description, status, priority)
VALUES
('Diseñar la interfaz del tablero', 'Today',
'high'),
('Preparar la reunión semanal', 'This week',
'medium'),
('Actualizar documentación técnica', 'Some day',
'low');

🔹 Consultar todas las tareas

SELECT * FROM tasks;

🔹 Consultar estados distintos

SELECT DISTINCT status FROM tasks;

🔹 Contar tarjetas del tablero según su estado

SELECT status, COUNT(*) AS total_tasks
FROM tasks
GROUP BY status
ORDER BY total_tasks DESC;

- Resultado de ejemplo:

status total_tasks
In progress 5
Today 3
Done 2
This week 2
Some day 1

🔹 Ordenar tareas por prioridad y fecha

SELECT * FROM tasks
ORDER BY priority ASC, created_at DESC;

🔹 Filtrar con condiciones lógicas

SELECT * FROM tasks
WHERE (status = 'Today' 
       OR status = 'In progress')
       AND priority IN ('top', 'high');

🔹 Actualizar tareas (cambiar estado)

UPDATE tasks
SET status = 'Done'
WHERE id = 1;

UPDATE tasks
SET description = 'Revisar interfaz final', 
    status = 'In progress'
WHERE id = 1;

🔹 Eliminar tareas completadas

DELETE FROM tasks
WHERE status = 'Done';

Archivo SQL completo para phpMyAdmin

Puedes copiar y pegar este código directamente en phpMyAdmin para crear la base de datos y tabla completas:

Archivo: kanban_setup.sql

-- Crear base de datos
CREATE DATABASE IF NOT EXISTS kanban_board;
USE kanban_board;

-- Crear tabla tasks
CREATE TABLE tasks (
    id INT AUTO_INCREMENT PRIMARY KEY,
    description TEXT,
    status ENUM('deleted',
                'Some day', 
                'This week', 
                'Tomorrow', 
                'Today', 
                'In progress',
                'Done') DEFAULT 'Some day',
    priority ENUM('top',
                  'high', 
                  'medium', 
                  'low') DEFAULT 'medium',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON 
    UPDATE CURRENT_TIMESTAMP
);

-- Insertar datos de ejemplo
INSERT INTO tasks (description, status, priority) VALUES
('Diseñar la interfaz del tablero', 'Today', 'high'),
('Preparar la reunión semanal', 'This week', 'medium'),
('Actualizar documentación técnica', 'Some day', 'low'),
('Implementar API REST', 'In progress', 'top'),
('Revisar código del frontend', 'Tomorrow', 'high');

Instrucciones para phpMyAdmin:

  1. Abre phpMyAdmin en tu navegador
  2. Ve a la pestaña “SQL”
  3. Copia y pega el código anterior
  4. Haz clic en “Continuar” o “Ejecutar”
  5. La base de datos kanban_board y la tabla tasks se crearán automáticamente con datos de ejemplo
